# Scholr 60-Second Demo Script

## Goal

Show that Scholr is a live academic product, not a fragile AI demo.

## 60-Second Story

1. Open the landing page.
   - Say: "Scholr is an AI-powered academic intelligence and research assistance platform for BTech students."
2. Click into the dashboard or Research module.
   - Say: "Instead of one blank chatbot, Scholr gives structured workflows for research, notes, and doubt solving."
3. Run a Research prompt like `Machine Learning for traffic prediction`.
   - Say: "When the provider is healthy, Scholr streams structured guidance in AI Mode."
4. Show the output panel and mode badge.
   - Say: "If quota is exhausted, Scholr does not break. It switches to Fallback Academic Mode or reuses a Cached Academic Response."
5. Open Notes or Doubt quickly.
   - Say: "The same resilient streaming flow works across all three core student modules."
6. End on the live repo or architecture docs.
   - Say: "The MVP is live on Vercel and Render, with provider diagnostics, cache, rate limiting, and graceful degradation built in."

## Mobile Demo Flow

1. Open the live landing page on iPhone.
2. Show the responsive header and module navigation.
3. Open Research or Doubt.
4. Trigger a prompt and show the streamed response.
5. Highlight the mode badge if fallback or cache is active.
6. End by showing that the output remains useful instead of empty.

## Desktop Demo Flow

1. Start on the landing page hero.
2. Open Research.
3. Run a prompt and show streaming.
4. Mention provider resilience while the answer loads.
5. Show dashboard history or switch to Notes/Doubt for a second example.
6. Close on the live URLs and architecture docs.
