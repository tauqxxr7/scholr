# Scholr Privacy Note

Scholr is owned by Tauqeer Bharde.

## What Scholr does

Scholr accepts prompts and questions to generate academic responses across Research, Notes, and Doubt workflows.

## Data handled

Scholr may handle:
- prompts and questions entered by users
- response metadata
- runtime diagnostics
- non-sensitive usage analytics
- optional local browser state

## What users should avoid submitting

Users should not submit passwords, private keys, financial secrets, sensitive personal data, health records, or confidential institutional documents.

## AI response processing

Inputs may be sent to external model providers to generate responses. Users remain responsible for checking whether they are comfortable with that processing before submitting sensitive material.

## Analytics and cookies

Scholr may use analytics, cookies, or local storage in a minimal way to understand usage, preserve interface state, and improve reliability. The current MVP is designed to avoid invasive tracking.

## Uploaded files

Document upload and retrieval flows are still a staged foundation. If enabled later, uploaded files may be temporarily stored or processed for parsing, chunking, embeddings, retrieval, and answer generation. Final retention and deletion policies will be tightened before wide release.

## Security note

Scholr is built with practical safeguards, but no internet-connected system can promise perfect security. Users should verify content and avoid submitting highly sensitive information.
