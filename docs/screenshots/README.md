# Screenshot Assets

This folder stores supporting product screenshots that should not clutter the top-level README.

Current structure:
- `desktop/` - desktop landing, dashboard, research, notes, doubt, and provider screenshots
- `mobile/` - mobile viewport proof assets kept for validation docs, not embedded in the README

README rule:
- keep one desktop hero screenshot near the top
- do not embed mobile screenshots in the README
- use diagrams for architecture and workflow instead of long image galleries

Suggested GIF workflow:
1. Record a 10-20 second flow: Research -> Notes -> Doubt.
2. Export as a compact GIF.
3. Place it at `docs/demo/demo.gif`.
4. Keep it optional and below the main technical content.
